## 1. 设计理念

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **新中式极简** | 融合中国传统美学元素（朱砂红、宣纸质感）与现代极简设计，不"大红大金"，而是含蓄内敛 |
| **内容优先** | 大留白、高对比度文字，确保数字信息的绝对可读性 |
| **触手可及** | 核心操作（新增记录）不超过 2 次点击到达 |
| **隐私感知** | 界面传递"安全"和"私密"的感受，避免社交化设计 |
| **情感化** | 通过细腻的触觉反馈和微动画，让"记账"不再枯燥 |

### 1.2 设计参考

| 参考应用 | 借鉴要素 |
|----------|----------|
| **Copilot Money** | 优雅的数据可视化、清晰的信息层级、现代化卡片布局 |
| **Budget Flow** | 极简录入流程、离线优先的设计理念、干净的列表布局 |
| **份子记账** | 中文语境下的人情记录流程、收/送双向交互 |
| **随手记** | 成熟的记账键盘设计、分类标签体系 |

## 2. 设计语言与品牌风格

### 2.1 色彩系统

#### 主色调

| 色彩名称 | 浅色模式 | 深色模式 | 用途 |

| **淡墨 (Text Secondary)** | `#8E8E93` | `#98989D` | 次要文字、描述 |

| 色彩名称 | 浅色模式 | 深色模式 | 用途 |

| 名称 | 浅色模式 | 深色模式 | 用途 |

| 名称 | 渐变值 | 用途 |

#### SwiftUI 颜色扩展代码

```swift
extension Color {
    struct Theme {
        // 主色
        let primary = Color("Primary")         // 朱砂红
        let background = Color("Background")   // 宣纸白
        let textPrimary = Color("TextPrimary") // 墨色
        let textSecondary = Color("TextSecondary") // 淡墨

        // 辅助色
        let received = Color("Received")       // 青竹 - 收到
        let sent = Color("Sent")               // 烟灰 - 送出
        let warning = Color("Warning")         // 琥珀
        let info = Color("Info")               // 浅丁香

        // 表面色
        let card = Color("CardBackground")
        let cardElevated = Color("CardElevated")
        let divider = Color("Divider")
    }

    static let theme = Theme()
}
```

### 2.2 字体系统

采用 iOS 系统字体，确保最佳渲染效果和 Dynamic Type 支持。

| 语义层级 | 字体 | 字号 | 字重 | 行高 | 用途 |
|----------|------|------|------|------|------|
| **LargeTitle** | SF Pro Display | 34pt | Bold | 41pt | 首页标题"礼尚记" |
| **Title1** | SF Pro Display | 28pt | Bold | 34pt | 页面标题 |
| **Title2** | SF Pro Display | 22pt | Bold | 28pt | 卡片标题、金额大字 |
| **Title3** | SF Pro Text | 20pt | Semibold | 25pt | 次级标题 |
| **Headline** | SF Pro Text | 17pt | Semibold | 22pt | 列表项标题 |
| **Body** | SF Pro Text | 17pt | Regular | 22pt | 正文内容 |
| **Callout** | SF Pro Text | 16pt | Regular | 21pt | 辅助文本 |
| **Subheadline** | SF Pro Text | 15pt | Regular | 20pt | 次要信息 |
| **Footnote** | SF Pro Text | 13pt | Regular | 18pt | 脚注、时间戳 |
| **Caption1** | SF Pro Text | 12pt | Regular | 16pt | 标签、图表标注 |
| **Caption2** | SF Pro Text | 11pt | Regular | 13pt | 极小提示文字 |

**金额专用排版：**

| 场景 | 字体 | 字号 | 字重 | 说明 |
|------|------|------|------|------|
| 仪表盘总额 | SF Pro Display | 42pt | Bold | 首页大数字，使用 Monospaced Digit |
| 卡片金额 | SF Pro Display | 28pt | Bold | 账本/记录卡片上的金额 |
| 列表金额 | SF Pro Text | 20pt | Semibold | 列表项中的金额 |

> **关键设计规则**：金额数字一律使用 `.monospacedDigit()` 修饰符，确保数字对齐。

### 2.3 间距与栅格系统

基于 **4pt 栅格**，所有间距为 4 的倍数：

| Token 名称 | 值 | 使用场景 |
|------------|-----|----------|
| `spacing.xs` | 4pt | 图标与文字间距、紧凑元素内部 |
| `spacing.sm` | 8pt | 相关元素间距、列表项内部间距 |
| `spacing.md` | 12pt | 组件内部填充 |
| `spacing.lg` | 16pt | 卡片内边距、列表项间距 |
| `spacing.xl` | 20pt | 区块间距 |
| `spacing.xxl` | 24pt | 页面区块间距 |
| `spacing.xxxl` | 32pt | 大区块间隔 |

**页面边距：**
- iPhone：水平 `16pt`
- iPad：水平 `20pt`（紧凑宽度）/ `24pt`（正常宽度）

### 2.4 圆角系统

| Token | 值 | 使用场景 |
|-------|-----|----------|
| `radius.sm` | 8pt | 按钮、标签、小卡片 |
| `radius.md` | 12pt | 输入框、列表组 |
| `radius.lg` | 16pt | 大卡片、Sheet |
| `radius.xl` | 20pt | 底部弹窗、全屏圆角 |
| `radius.full` | capsule | 胶囊按钮、标签 |

### 2.5 阴影系统

| Token | 值 | 使用场景 |
|-------|-----|----------|
| `shadow.sm` | 0 1pt 3pt rgba(0,0,0,0.06) | 轻微浮起效果 |
| `shadow.md` | 0 4pt 12pt rgba(0,0,0,0.08) | 卡片默认阴影 |
| `shadow.lg` | 0 8pt 24pt rgba(0,0,0,0.12) | 浮层、弹窗 |

> **深色模式下**：不使用阴影，改用微弱边框（1pt, `#3A3A3C`）区分层级。

### 3.1 导航结构

采用 **底部 Tab 导航** 作为主导航，共 4 个 Tab：

```
┌─────────────────────────────────────────────────────┐
│                    内容区域                            │
│                                                       │
│                                                       │
│                                                       │
│                                                       │
```

| Tab | 图标 (SF Symbol) | 标题 | 功能 |
|-----|------------------|------|------|
| 首页 | `house.fill` | 首页 | 仪表盘、最近记录、快速操作 |

**快速录入入口：** 首页和账本页均有悬浮的 "+" 按钮，点击弹出录入方式选择（手动/OCR/语音）。

### 3.3 核心用户流程

**流程一：手动录入一笔记录**

```
首页/账本 → 点击 "+" FAB
→ 选择"手动录入"
→ Sheet 弹出录入表单
   ├── 选择 收到/送出（Segmented Control）
   ├── 输入金额（自定义大数字键盘）
   ├── 选择/输入联系人（自动联想）
   │   └── 显示该联系人历史往来摘要
   ├── 选择事件类型（横向滚动标签）
   ├── 选择日期（DatePicker）
   ├── 选择所属账本（Picker）
   └── 备注（可选，TextEditor）
→ 点击"保存"
→ 触觉反馈 + Toast 提示"记录成功"
→ 回到列表，新记录带入场动画
```

**流程二：OCR 扫描录入**

```
首页/账本 → 点击 "+" FAB
→ 选择"拍照识别"
→ 全屏打开摄像头
   ├── 辅助框引导对准纸质礼单
   └── 支持从相册选取
→ 拍摄/选取后进入识别处理
   └── 加载动画 + "正在识别..."
→ 进入校对界面
   ├── 左侧：原图切片
   ├── 右侧：识别结果（可编辑）
   ├── 绿色 ✓ 标记高置信度
   ├── 橙色 ⚠ 标记低置信度（需关注）
   └── 可逐条修改或删除
→ 选择目标账本
→ 点击"全部保存"
→ 批量创建成功提示
```

**流程三：语音快速记账**

```
首页/账本 → 点击 "+" FAB
→ 选择"语音录入"
→ Sheet 弹出语音输入界面
   ├── 大麦克风按钮，按住说话
   ├── 实时展示识别文字
   └── 示例提示："请说出：张三 结婚 八百八十八"
→ 松开按钮，系统解析
   ├── 展示解析结果：
   │   联系人：张三
   │   事件：结婚
   │   金额：¥888
   │   方向：送出
   └── 用户确认或修改
→ 点击"保存"
→ 成功反馈
```

**智能提醒卡片：**
- 出现在列表底部
- 淡黄色背景，灯泡图标

#### 自定义金额键盘设计

- **键盘按钮尺寸**：最小 64x52pt，确保大拇指轻松点击
- **数字键**：白色背景，大号字体（24pt Bold），点击带触觉反馈（`.light`）
- **保存键**：朱砂红背景，白色文字，点击带 `.medium` 触觉反馈
- **快捷金额键**（666/888 等）：出现在键盘上方，横向滚动的标签条

## 5. 交互规范

### 5.1 转场动画

| 场景 | 转场方式 | 动画参数 |
|------|----------|----------|
| Tab 切换 | 无动画（即时切换） | - |
| 进入二级页面 | NavigationStack Push | 系统默认滑入 |
| 录入表单 | Sheet (Detent: .large) | 从底部弹出 |
| OCR 扫描 | FullScreenCover | 从底部弹出，全屏 |
| 删除确认 | Alert / ConfirmationDialog | 系统默认 |
| Toast 提示 | 自定义动画 | 从顶部滑入，2秒后自动消失 |
| 新记录出现 | List 插入动画 | `.spring(response: 0.4)` |

### 5.2 触觉反馈规范

| 操作 | 反馈类型 | UIKit API |
|------|----------|-----------|
| 数字键盘按键 | 轻触 | `UIImpactFeedbackGenerator(style: .light)` |
| 保存成功 | 成功 | `UINotificationFeedbackGenerator().notificationOccurred(.success)` |
| 删除操作 | 警告 | `UINotificationFeedbackGenerator().notificationOccurred(.warning)` |
| 滚动选择器 | 选择变更 | `UISelectionFeedbackGenerator().selectionChanged()` |
| 金额数字滚动 | 轻触 | `UIImpactFeedbackGenerator(style: .light)` |
| 长按菜单弹出 | 中等 | `UIImpactFeedbackGenerator(style: .medium)` |
| OCR 识别完成 | 成功 | `UINotificationFeedbackGenerator().notificationOccurred(.success)` |
| 错误/失败 | 错误 | `UINotificationFeedbackGenerator().notificationOccurred(.error)` |

#### SwiftUI 触觉反馈扩展

```swift
extension View {
    func hapticLight() -> some View {
        self.sensoryFeedback(.impact(flexibility: .rigid, intensity: 0.5), trigger: true)
    }

    func hapticSuccess() -> some View {
        self.sensoryFeedback(.success, trigger: true)
    }
}
```

### 5.3 手势操作

| 手势 | 场景 | 效果 |
|------|------|------|
| **左滑** | 记录列表项 | 显示"编辑"和"删除"操作 |
| **长按** | 账本卡片 | 弹出上下文菜单（编辑/归档/导出/删除） |
| **下拉** | 列表顶部 | 触发数据刷新（动画+触觉反馈） |
| **左右滑动** | 首页概览卡片 | 切换时间范围（本月/本年/全部） |
| **捏合** | 统计图表 | 缩放时间范围 |
| **点击空白** | 键盘展开时 | 收起键盘 |

### 6.2 LSJButton（按钮组件）

定义三种按钮样式：

| 样式 | 外观 | 使用场景 |
|------|------|----------|
| **Primary** | 朱砂红背景 + 白色文字 | 主要操作（保存、确认） |
| **Secondary** | 白色背景 + 朱砂红文字 + 朱砂红边框 | 次要操作（取消、编辑） |
| **Text** | 无背景 + 朱砂红文字 | 文字链接（查看全部、更多） |

```swift
enum LSJButtonStyle {
    case primary, secondary, text
}

struct LSJButton: View {
    let title: String
    let style: LSJButtonStyle
    let action: () -> Void

    var body: some View {
        Button(action: action) {
            Text(title)
                .font(.headline)
                .frame(maxWidth: .infinity)
                .padding(.vertical, 14)
                .background(backgroundColor)
                .foregroundStyle(foregroundColor)
                .clipShape(RoundedRectangle(cornerRadius: 12))
                .overlay(
                    RoundedRectangle(cornerRadius: 12)
                        .strokeBorder(borderColor, lineWidth: style == .secondary ? 1 : 0)
                )
        }
    }

    private var backgroundColor: Color {
        switch style {
        case .primary: return Color.theme.primary
        case .secondary: return Color.theme.card
        case .text: return .clear
        }
    }

    private var foregroundColor: Color {
        switch style {
        case .primary: return .white
        case .secondary, .text: return Color.theme.primary
        }
    }

    private var borderColor: Color {
        switch style {
        case .secondary: return Color.theme.primary
        default: return .clear
        }
    }
}
```

### 7.3 深色模式

**所有颜色均在 Assets.xcassets 中定义 Any Appearance / Dark Appearance 两套色值**，确保自动切换。

深色模式设计原则：

| 原则 | 说明 |
|------|------|
| 降低对比度 | 不使用纯黑(#000000)，改用深灰(#1C1C1E) |
| 提亮主色 | 朱砂红从 #C04851 提亮为 #E05A63 |
| 取消阴影 | 改用微弱边框（1pt）区分层级 |
| 保持品牌感 | 朱砂红仍为主视觉色，不变灰 |

### 7.4 动态字体 (Dynamic Type)

所有文字使用 SwiftUI 语义字体（`.title`, `.body`, `.caption` 等），自动支持 Dynamic Type。

关键注意事项：

```swift
// 正确：使用语义字体
Text("¥888").font(.title2.bold().monospacedDigit())

// 错误：硬编码字号
Text("¥888").font(.system(size: 22, weight: .bold))
```

对于需要固定尺寸的区域（如金额键盘），使用 `@ScaledMetric`：

```swift
@ScaledMetric(relativeTo: .title) private var keySize: CGFloat = 52
```

### 7.5 无障碍 (Accessibility)

| 要求 | 实现方式 |
|------|----------|
| 色彩对比度 >= 4.5:1 | 所有文字/背景组合通过 WCAG AA 标准 |
| VoiceOver 支持 | 所有交互元素添加 `.accessibilityLabel` |
| 金额朗读 | 金额添加 `.accessibilityValue("八百八十八元")` |
| 大字体模式 | 使用语义字体 + ScrollView 应对溢出 |
| 减弱动态效果 | 检测 `accessibilityReduceMotion` 禁用动画 |
| 按钮最小可触区域 | 44x44pt |

```swift
// VoiceOver 示例
Text("¥888")
    .accessibilityLabel("金额")
    .accessibilityValue("八百八十八元")

// 方向指示
Image(systemName: "arrow.up.circle.fill")
    .accessibilityLabel("送出")
```
